concepts of mcp
    client-server architecture

    local MCP : typically server 1 mcp connect 

    remote MCP : use streamable HTTP transport, typically server mutiple MCP clients

    key participants:
        mcp hosts: ai application that coordinates or manage mutiple mcp clients
        mcp client: component that maintains a connection to the mcp server and obtains context from mcp server for the mcp host to use 
        mcp server: a program that provides context to mcp clients 

    layers:
        data layers: 
            defines json-rpc based protocal for client-server communication 
            - lifecycle management: Handles connection initialization, capability negotiation, and connection termination between clients and servers
            - server features : Enables servers to provide core functionality
            - client features : Enables servers to ask the client to sample from the host LLM
            - utility features : additional capabilities : noti for real-time updates, progress tracking 


            data layer protocol:
                defining the schema and semantic between MCP clients and MCP servers 
                lifecycle management :
                primitives :
                    These primitives specify the types of contextual information that can be shared with AI applications
                     and the range of actions that can be performed.
                    servers:
                        - tools : Executable functions that AI applications can invoke to perform actions
                        - resources : data resources (file contents, database, API responses)
                        - prompts : Reusable templates that help structure interactions with language models
                    clients:
                        - sampling : allow server to request ai model completion
                        - excitiation : allow server to request additional information from the user 
                        - logging : enabling server to send log messages to client 
            

            





        transport layer:
            defines communication mechanisms and channels (connection establishment, secure commnuication  , auth, message framing ?)
            - stdio transport: direct process communication between local processes on the same machine
            - streamable HTTP transport : Uses HTTP POST for client-to-server messages with optional Server-Sent Events for streaming capabilities.
                         This transport enables remote server communication and supports standard HTTP authentication methods.
